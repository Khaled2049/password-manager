# Best Practices for Users

## DO

```
✓ Use a strong, unique master password (12+ characters)
✓ Store master password in a physical safe place
✓ Lock vault when not in use (especially on shared devices)
✓ Enable device encryption (FileVault, BitLocker, etc.)
✓ Keep browser and OS updated
✓ Use HTTPS sites only
✓ Verify domain before entering password
✓ Use auto-lock timeout features
✓ Regularly backup vault (export encrypted copy)
```

## DON'T

```
✗ Reuse master password from other services
✗ Share master password with anyone
✗ Store master password in browser password manager
✗ Use dictionary words or personal info in password
✗ Leave vault unlocked on public/shared computers
✗ Enter password on unsecured WiFi without VPN
✗ Trust password recovery claims (impossible with zero-knowledge)
✗ Use weak passwords like "password123"
```

## Password Strength Guidelines

- **Minimum**: 8 characters (enforced by system)
- **Recommended**: 12+ characters
- **Strong**: Mix of uppercase, lowercase, numbers, and symbols
- **Avoid**: Dictionary words, personal information, common patterns

## Security Checklist

- [ ] Master password is 12+ characters
- [ ] Master password is unique (not reused elsewhere)
- [ ] Device has full-disk encryption enabled
- [ ] Browser and OS are up to date
- [ ] Antivirus/anti-malware is installed and updated
- [ ] Vault is locked when not in use
- [ ] Master password is stored securely offline
- [ ] Regular backups of encrypted vault are made
